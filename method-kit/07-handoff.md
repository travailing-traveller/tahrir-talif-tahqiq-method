# 07 — Handoff

Use this to compact a session into the project ledger.

## Do not duplicate everything

The folder is the durable ledger. The handoff should point to canonical files rather than repeat them unnecessarily.

## Handoff sections

1. Date
2. Session purpose
3. Work completed
4. Files changed
5. Decisions made
6. Questions opened or resolved
7. Claims added or verified
8. Source issues
9. Terminology issues
10. Verification status
11. Risks
12. Next action

## Update required ledgers

Where applicable, update:

- `planning/STATE.md`
- `planning/DECISIONS.md`
- `planning/QUESTIONS.md`
- `planning/RISKS.md`
- `planning/SOURCE_LEDGER.md`
- `planning/CLAIMS_LEDGER.md`
- `planning/TERMINOLOGY.md`
- `planning/UNITS.md`
