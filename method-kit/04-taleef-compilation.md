# 04 — Taʾlīf Compilation

Use this to compose, translate, annotate, summarise, or arrange one defined unit from retrieved sources.

## Inputs required

- project foundation;
- unit ID;
- source handle(s);
- source span(s);
- permitted operation;
- output register or base layer;
- terminology rules;
- acceptance criteria.

## Rules

- Work only on the defined unit.
- Use retrieved source spans, not model memory.
- Preserve source handles.
- Mark translation, explanation, paraphrase, inference, and application separately.
- Record any new claim in `planning/CLAIMS_LEDGER.md` or unit notes.
- Record uncertainty instead of resolving it by invention.

## Unit output should include

- unit ID;
- source handles;
- source span(s);
- base output;
- notes;
- claims;
- terminology issues;
- uncertainties;
- proposed verification checks.
