# sprints/

Planned stages of work (marḥala). Each `NNN-slug/` folder groups a batch of units
and carries the quartet: `requirements.md`, `blueprint.md`, `acceptance.md`,
`handoff.md`.

Use sprints when the work spans many units or multiple layers (a tafsīr pipeline,
a multi-volume taḥqīq). A single-unit work does not need them — point `STATE.md`
straight at the unit. Units live in `../units/` keyed by locus; a sprint
references the units it covers and the layer it adds. Copy `../templates/sprint/`
to start a new one.
